# ActionCard

An actionable card with a title, description, optional thumbnail and completion badge, plus a primary action button. The actionable-card sibling to `@m-next/card` (which displays a record).

The audit calls out a `ContentCard → ActionCard` rename. The canonical export is now `ActionCard`; the legacy default export `ContentCard` and the package name `@m-next/content-card` are preserved so existing imports keep working.

## When to use it

- Onboarding tiles, demo launchers, "next step" suggestion cards.
- Lesson / module cards with a primary CTA and a time-to-complete chip.
- Marketing-style content tiles inside a product surface.

## When NOT to use it

- Record summaries with field/value pairs — use `@m-next/card`.
- Dashboard KPI tiles — use `@m-next/insight-card`.
- A vanilla container with arbitrary children — use a Container directly.
- An entirely-clickable surface — wrap the ActionCard in an `<a>` or `<Link>`. The card's `onClick` is wired to the inner button, not the surface.

## Anatomy

A vertical card holding:

1. **Thumbnail** — the top slot. Either an `<img>` from `thumbnail`, or a grey placeholder.
2. **Completion badge** (optional) — absolute-positioned in the top-right when `isComplete && icon` are both set.
3. **Bottom section** — title (single-line clamp) + description (two-line clamp) + CTA row.
4. **CTA row** — a ghost `@m-next/button` on the left, an optional time-to-complete `Pill` on the right.

## Tokens consumed

- `colors.white` — card background
- `colors.grey.light` — border
- `colors.grey.lighter` — thumbnail placeholder
- `colors.grey.darkest` — title text
- `colors.grey.base` — description text
- `colors.green.base` — default completion-icon color

All hex literals and `var(--Grey-*)` references in the legacy styles have been migrated to `@m-next/tokens`.

## API

See the README for the full prop table. Highlights:

| Prop | Type | Notes |
|------|------|-------|
| `title` | string | Heading text |
| `description` | string | Body copy, two-line clamp |
| `buttonText` | string | Label on the inner button (default "Launch Demo") |
| `onClick` | function | Click handler for the inner button |
| `thumbnail` | string | Optional image URL |
| `timeToComplete` | string \| number | Renders `~Nmin` Pill |
| `isComplete` + `icon` | boolean + string | Renders a corner badge |

## Accessibility

- Card root is a `<div>`. Native attributes (`role`, `tabIndex`, `aria-*`) spread through.
- The action button is a real `@m-next/button` — keyboard and focus semantics are inherited.
- Thumbnail `alt` defaults to `title || 'thumbnail'`. Override with `alt=""` for purely decorative imagery.

## Migration from `@m-one/content-card`

| Was | Now | Status |
|-----|-----|--------|
| `ContentCard` only | `ActionCard` canonical, `ContentCard` legacy alias / default export | Backwards-compatible |
| `id` required | optional, auto-generated | Backwards-compatible |
| `forwardRef` prop | React forwardRef API (`ref={…}`) | Soft shim — warns once |
| `iconColor='green'` default | `colors.green.base` default | Internal-only |
| Hex literals in CSS | `@m-next/tokens.colors.*` | Internal-only |
| `isV4Design`, `isMobile`, `legacyClass`, `displayAuto`, `compactStyle`, `hidden` | — | Silently ignored |

## Edge cases

- **`isComplete={true}` without `icon`** — the badge does not render. Both are required for the corner overlay.
- **`thumbnail` plus `title`** — the image `alt` falls back to `title`; pass `alt=""` via rest-spread if the image is purely decorative.
- **`timeToComplete={0}`** — the Pill does not render (falsy check). Pass `1` or higher if you need to show a `~0min` indicator.
- **`forwardRef` prop AND `ref` prop** — both are attached to the wrapper. Prefer `ref`; the legacy prop is shimmed for transition.
